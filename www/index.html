🔑 Where do I find my remote code?
  1. Turn on your Freebox Player (TV) and press the Free (home) button on the physical remote.
  2. Go to Réglages → Système → Informations Freebox Player et Server.
  3. Read the Code télécommande réseau (8 digits), paste it above and click Save.
  4. Your phone or computer must be on the same Wi-Fi as the Freebox Player.

❓ Help
What is this app?

A Workshop-DIY educational web app. Explore, create, and innovate!

How do I change the theme?

Open Settings (⚙️) and pick a theme from the dropdown.

How do I change the language?

Open Settings (⚙️) and pick your language. Arabic enables RTL automatically.

Is my data private?

Yes. Everything runs locally in your browser. No data is sent anywhere.

FreeZap loads but no buttons work (GitHub Pages / HTTPS). How do I unblock it?

Browsers block HTTP calls from HTTPS pages (mixed content). In desktop Chrome: open chrome://settings/content/insecureContent, under "Allowed to show insecure content" click Add and enter the page domain (e.g. https://abourdim.github.io), then reload. This trick does NOT work on Safari iOS or Chrome Android — on phones, use a local HTTP server instead (see next question).

How do I use FreeZap on my phone?

Run ./serve.sh from the project folder on a computer that stays on your Wi-Fi (Mac, Raspberry Pi, NAS). It prints a URL like http://192.168.1.15:8000/ — open that on your phone in Safari or Chrome (same Wi-Fi) and tap "Add to Home Screen". Everything is plain HTTP so no mixed-content block.

Does FreeZap work with my Freebox Pop / Delta / Ultra?

No. FreeZap uses the legacy unauthenticated HTTP remote API that only exists on the Freebox Revolution (v6) Player. Pop, Delta and Ultra use a different token-authenticated API that FreeZap does not implement.

Can I control FreeZap with the keyboard?

Yes. Arrows navigate, Enter is OK, Esc/Backspace is Back, Space is Play/Pause, + and − are volume, m is Mute, h is Home, i is Info, and 0–9 send the matching digit.

Does FreeZap need an internet connection?

No. Once the page is loaded, FreeZap only talks to your Freebox on the local network. Your phone/computer and the Freebox Player just need to be on the same Wi-Fi — that Wi-Fi doesn't even need internet access.

What is "long press"?

Hold any on-screen button for more than 500 ms and the key is sent with &long=true, matching the behaviour of holding the physical remote (useful for fast volume / channel scrolling).

What's the colored pill at the top right?

It's the live Freebox status. Possible states: 🟡 No code — you haven't saved your remote code yet. ⚪ Checking… — the initial LAN probe is running. 🟢 Freebox reachable — the box responds on the LAN. 🔴 Freebox unreachable — network / DNS / wrong Wi-Fi. 🟢 ✓ Ns ago — last button press succeeded. 🔴 ✗ Ns ago — last button press failed. Re-probes every 20 seconds automatically.

Can I see the current channel / volume / power state in FreeZap?

Yes — experimentally, via the optional freezap-agent. It's a tiny Python 3 bridge (stdlib only, ~400 lines) that pairs with the authenticated Freebox OS API and exposes real-time player state to FreeZap over http://localhost:8766/. When it's running and paired, the pill upgrades to e.g. 📺 TF1 · 🔊 Vol 42 instead of just "Reachable". See freezap-agent/README.md in the repo for the setup.

Why is the FreeZap Agent marked "experimental"?

Because its pairing flow hasn't been verified end-to-end on a live Freebox Revolution yet. The HTTP protocol works correctly (POST /api/v6/login/authorize/ returns a valid track, and GET …/{id} reports status: pending), but in 4 real-world attempts against a Revolution (r1, firmware api_version 15.0) the physical validation step never completed — no prompt appeared on the front-panel OLED, and no pending request showed up under Freebox OS → Gestion des accès → Applications. Likely causes: aging OLED hardware, firmware quirk, or the auth UI having moved. FreeZap still works perfectly without the agent — it falls back to the passive reachability pill. See the "Known limitation" section in freezap-agent/README.md for workarounds (including reusing an existing app_token from Home Assistant, Jeedom, or freebox-api).
